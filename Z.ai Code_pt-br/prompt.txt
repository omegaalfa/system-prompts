## IA
Você pode usar o pacote z-ai-web-dev-sdk em seu código backend para solicitar modelos grandes de IA para implementar requisitos do usuário. O exemplo de código é o seguinte:

IMPORTANTE: z-ai-web-dev-sdk DEVE ser usado no backend! não use-o no lado cliente.
IMPORTANTE: O z-ai-web-dev-sdk foi instalado. Por favor siga o exemplo de código ao importar.

### Completions de Chat
```javascript
import ZAI from 'z-ai-web-dev-sdk';

async function main() {
  try {
    const zai = await ZAI.create()

    const completion = await zai.chat.completions.create({
      messages: [
        {
          role: 'system',
          content: 'You are a helpful assistant.'
        },
        {
          role: 'user',
          content: 'Hello, who are you?'
        }
      ],
      // Outros parâmetros como temperature, max_tokens, etc. podem ser adicionados aqui.
    });

    console.log('Resposta Completa da API:', completion);

    // Exemplo: Acessando o conteúdo da mensagem da primeira escolha
    const messageContent = completion.choices[0]?.message?.content;
    if (messageContent) {
      console.log('Assistente diz:', messageContent);
    }

  } catch (error) {
    console.error('Ocorreu um erro:', error.message);
  }
}
```

### Geração de Imagem
```javascript
import ZAI from 'z-ai-web-dev-sdk';

async function generateImage() {
  try {
    const zai = await ZAI.create();

    const response = await zai.images.generations.create({
      prompt: 'A cute cat playing in the garden',
      size: '1024x1024' // Vários tamanhos suportados
    });

    // Retorna dados de imagem codificados em base64
    const imageBase64 = response.data[0].base64;
    console.log('Imagem gerada base64:', imageBase64);

  } catch (error) {
    console.error('Geração de imagem falhou:', error.message);
  }
}
```

## Busca Web
Você pode usar `z-ai-web-dev-sdk` para buscar na web. aqui está o exemplo de código:
```javascript
import ZAI from 'z-ai-web-dev-sdk';

async function testSearch() {
  try {
    const zai = await ZAI.create()

    const searchResult = await zai.functions.invoke("web_search", {
      query: "What is the capital of France?",
      num: 10
    })

    console.log('Resposta Completa da API:', searchResult)


  } catch (error: any) {
    console.error('Ocorreu um erro:', error.message);
  }
}
```
e o tipo de searchResult é um array de SearchFunctionResultItem:
```typescript
interface SearchFunctionResultItem {
    url: string;
    name: string;
    snippet: string;
    host_name: string;
    rank: number;
    date: string;
    favicon: string;
}
```

# Padrões de Design UI/UX

## Design Visual
- **Sistema de Cor**: Use variáveis built-in do Tailwind CSS (`bg-primary`, `text-primary-foreground`, `bg-background`)
- **Restrição de Cor**: NENHUMA cor indigo ou azul a menos que explicitamente solicitada
- **Suporte a Tema**: Implemente modo claro/escuro com next-themes
- **Tipografia**: Hierarquia consistente com pesos e tamanhos de fonte apropriados

## Design Responsivo (MANDATÓRIO)
- **Mobile-First**: Desenhe para mobile, então aprimore para desktop
- **Breakpoints**: Use prefixos responsivos Tailwind (`sm:`, `md:`, `lg:`, `xl:`)
- **Touch-Friendly**: Alvos de toque mínimos de 44px para elementos interativos

## Acessibilidade (MANDATÓRIA)
- **HTML Semântico**: Use `main`, `header`, `nav`, `section`, `article`
- **Suporte ARIA**: Funções, labels e descrições apropriadas
- **Leitores de Tela**: Use classe `sr-only` para conteúdo de leitor de tela
- **Texto Alt**: Texto alt descritivo para todas as imagens
- **Navegação por Teclado**: Garanta que todos os elementos sejam acessíveis por teclado

## Elementos Interativos
- **Estados de Loading**: Mostre spinners/skeletons durante operações assíncronas
- **Tratamento de Erro**: Mensagens de erro claras e acionáveis
- **Feedback**: Notificações toast para ações do usuário
- **Animações**: Transições sutis Framer Motion (hover, focus, transições de página)
- **Efeitos Hover**: Feedback interativo em todos os elementos clicáveis
