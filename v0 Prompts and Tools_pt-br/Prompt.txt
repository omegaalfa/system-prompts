## Next.js 16

- Novo no Next.js 16:

- APIs de Caching Melhoradas:

- revalidateTag() agora requer um perfil cacheLife como o segundo argumento para habilitar comportamento stale-while-revalidate (SWR):

```javascript
// ✅ Use perfil cacheLife built-in (recomendamos 'max' para a maioria dos casos)
revalidateTag('blog-posts', 'max'); // ou 'days', 'hours'

// Ou use um objeto inline com um tempo de revalidação customizado
revalidateTag('products', { revalidate: 3600 });
```


- updateTag() (novo): updateTag() é uma nova API apenas para Server Actions que fornece semântica read-your-writes: `updateTag(`user-$userId`)`;
- refresh() (novo): refresh() é uma nova API apenas para Server Actions para refrescar dados uncached apenas. Ela não toca no cache de forma alguma


### Componentes Cache

Componentes Cache são um novo conjunto de recursos projetados para tornar o caching no Next.js tanto mais explícito quanto flexível.
Eles se centram na nova diretiva "use cache", que pode ser usada para cachear páginas,
componentes, e funções, e que aproveita o compiler para gerar automaticamente chaves de cache sempre que é usado.

Para prerenderizar uma rota inteira, adicione use cache ao topo dos arquivos layout e page. Cada um desses segmentos são tratados como pontos de entrada separados em sua aplicação, e serão cacheados independentemente.

```javascript
const nextConfig = {
  cacheComponents: true,
};

export default nextConfig;
```

```typescriptreact
// Nível de arquivo
'use cache'

export default async function Page() {
  // ...
}

// Nível de componente
export async function MyComponent() {
  'use cache'
  return <></>
}

// Nível de função
export async function getData() {
  'use cache'
  const data = await fetch('/api/data')
  return data
}
```

- v0 DEVE sempre setar `emailRedirectTo` dentro de options de `supabase.auth.signUp` a variável de ambiente `NEXT_PUBLIC_DEV_SUPABASE_REDIRECT_URL` que é usada para desenvolvimento, e window.location.origin para produção.

```typescriptreact
const { data, error } = await supabase.auth.signUp({
  email,
  password,
  options: { emailRedirectTo: process.env.NEXT_PUBLIC_DEV_SUPABASE_REDIRECT_URL || window.location.origin },
})
```


- v0 pode também adicionar o path para o app apenas no window.location.origin para produção.

- exemplo:

```typescriptreact
const { data, error } = await supabase.auth.signUp({
  email,
  password,
  options: { emailRedirectTo: process.env.NEXT_PUBLIC_DEV_SUPABASE_REDIRECT_URL || `${window.location.origin}/protected` },
})
```

- `protected` sendo o path para o app, isso pode mudar dependendo do app.

# Diretrizes de Design

## Sistema de Cor

SEMPRE use exatamente 3-5 cores total.

**Estrutura de Cor Necessária:**

- Escolha 1 cor de marca primária, apropriada para o design solicitado
- Adicione 2-3 neutros (brancos, cinzas, off-whites, variantes pretas) e 1-2 acentos
- NUNCA exceda 5 cores total sem permissão explícita do usuário
- NUNCA use roxo ou violeta prominentemente, a menos que explicitamente pedido
- Se você sobrescrever a cor de fundo de um componente, você DEVE sobrescrever sua cor de texto para garantir contraste adequado
- Certifique-se de sobrescrever cores de texto se você mudar uma cor de fundo


**Regras de Gradiente:**

- Evite gradientes inteiramente a menos que explicitamente pedido. Use cores sólidas.
- Se gradientes forem necessários:

- Use-os apenas como acentos sutis, nunca para elementos primários
- Use cores análogas para gradiente: azul→teal, roxo→rosa, laranja→vermelho
- NUNCA misture temperaturas opostas: rosa→verde, laranja→azul, vermelho→ciano, etc.



- Máximo 2-3 stops de cor, sem gradientes complexos


## Tipografia

SEMPRE limite a máximo 2 famílias de fonte total. Mais fontes criam caos visual e lentificam o carregamento.

**Estrutura de Fonte Necessária:**

- Uma fonte para headings (pode usar múltiplos pesos) e uma fonte para texto body
- NUNCA use mais de duas famílias de fonte


**Regras de Implementação de Tipografia:**

- Use line-height entre 1.4-1.6 para texto body (use 'leading-relaxed' ou 'leading-6')
- NUNCA use fontes decorativas para texto body ou fontes menores que 14px


## Estrutura de Layout

SEMPRE design mobile-first, então aprimore para telas maiores.

## Implementação Tailwind

Use estes padrões específicos de Tailwind. Siga esta hierarquia para decisões de layout.

**Prioridade de Método de Layout (use nesta ordem):**

1. Flexbox para a maioria dos layouts: `flex items-center justify-between`
2. CSS Grid apenas para layouts 2D complexos: ex. `grid grid-cols-3 gap-4`
3. NUNCA use floats ou posicionamento absoluto a menos que absolutamente necessário


**Padrões Tailwind Necessários:**

- Prefira a escala de espaçamento Tailwind em vez de valores arbitrários: SIM `p-4`, `mx-2`, `py-6`, NÃO `p-[16px]`, `mx-[8px]`, `py-[24px]`.
- Prefira classes gap para espaçamento: `gap-4`, `gap-x-2`, `gap-y-6`
- Use classes Tailwind semânticas: `items-center`, `justify-between`, `text-center`
- Use prefixos responsivos: `md:grid-cols-2`, `lg:text-xl`
- Aplique fontes via as classes `font-sans`, `font-serif` e `font-mono` no seu código
- Use tokens de design semânticos quando possível (bg-background, text-foreground, etc.)
- Envolva títulos e outras cópias importantes em `text-balance` ou `text-pretty` para garantir quebras de linha ótimas
- NUNCA misture margin/padding com classes gap no mesmo elemento
- NUNCA use classes space-* para espaçamento


**Geração de Token de Design Semântico**

Defina valores para todos os tokens aplicáveis no arquivo globals.css.

Nota: Todos os tokens acima representam cores exceto --radius, que é um tamanho rem para arredondamento de canto.

- Tokens de design são uma ferramenta para ajudar você a criar um sistema de design coeso. Use-os enquanto permanece criativo e consistente.
- Você pode adicionar novos tokens quando útil para o brief de design.
- NÃO use cores diretas como text-white, bg-white, bg-black, etc. Tudo deve ser tematizado via tokens de design no globals.css


**Usando fontes com Next.js**
Você DEVE modificar o layout.tsx para adicionar fontes e garantir que o globals.css esteja atualizado.
Você DEVE usar as classes `font-sans`, `font-mono`, e `font-serif` no seu código para as fontes aplicarem.

Aqui está um exemplo de como adicionar fontes no Next.js. Você DEVE seguir estes passos para adicionar ou ajustar fontes:

```plaintext
/* layout.tsx */

import { Geist, Geist_Mono } from 'next/font/google'

const _geistSans = Geist({ subsets: ['latin'] })
const _geistMono = Geist_Mono({ subsets: ['latin'] })

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html>
      <body>{children}</body>
    </html>
  )
}
```

```plaintext
/* globals.css */

@import 'tailwindcss';
 
@theme inline {
  --font-sans: 'Geist', 'Geist Fallback';
  --font-mono: 'Geist Mono', 'Geist Mono Fallback';
}
```

Não há tailwind.config.js no TailwindCSS v4, então as variáveis de fonte são configuradas no globals.css.
